# Under-the-Hoodie

Public showcase of the AI agent operating model behind Henry Tano's consulting practice.

## What This Is

A single-page static website explaining how this practice organises itself using AI agents —
the system structure, the operating philosophy, and real examples of it in use.

**Live at:** tigeroid.github.io/under-the-hoodie (once deployed)
**Repo:** github.com/tigeroid/under-the-hoodie (standalone public repo, separate from portfolio)

## Purpose

Henry finds it hard to explain the operating model to others. This is the artifact that does it.

**Audiences:**
- Mezzanine Group stakeholders — understand what Henry is building
- Potential clients — see the operating model in action before engaging
- Other builders — share ideas, invite critique and collaboration
- Public — general interest in AI agent systems

## Content Structure

1. **The problem** — why most AI setups are tool piles, not systems
2. **The operating model** — Tanuki + Kitsune + project agents, with a diagram
3. **The philosophy** — honest, capacity-aware, portfolio-first
4. **In practice** — real examples (LOE generation POC, code review workflow)
5. **What we're building toward** — invitation for feedback and collaboration

## Technical Setup

- **System:** Claude Code native (no BMAD)
- **Stack:** Plain HTML/CSS — no framework
- **Hosting:** GitHub Pages on tigeroid account
- **Source of truth:** This repo is canonical. No Notion mirror.

## Working Principles

- Content over design complexity
- Honest about what's built vs what's in progress
- The philosophy should be evident in how the page reads, not just what it says
- Ship at 80% — a good page that exists beats a perfect page that doesn't
